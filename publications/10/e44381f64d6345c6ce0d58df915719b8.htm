Постановка завдання:

1.   Створити клас відкриття та закриття з’єднання з базою даних.

2.   Створити таблиці бази даних.

Завдання 6.

Створити виключення на помилки при виконанні SQL-запитів і обробити виключення.

Для цього завдання створимо новий клас виключення, який буде успадковуватися від  класу Exception.

class WrongDateException extends Exception {

  public function __construct($message = "Неверный формат даты", $code = 0, Exception $previous = null) {

        parent::__construct($message, $code, $previous);

    }

}

В його конструкторі задаємо параметри за замовчуванням. Це виключення буде викидатись, коли користувач буде вводите дату неправильного формату, тим самим ми зможемо захистити нашу базу даних від помилкових даних.

Тепер обработаємо це виключення. Встановимо блок try-catch в методі реєстрації нового користувача.

 

 

try {

    if (strlen($_POST['birthDay']) != 12)

        throw new WrongDateException();

} catch (WrongDateException $ex) {

        $this->failMsg = "Неверный формат даты!";

        return;

}

Тепер користувач не зможе зареєструватись, якщо він введе дату неправильної довжина, бо він побачить про це повідомлення, а сам процес реєстрації буде перерваний.

Відповіді на контрольні запитання

1.     Переваги  та обмеження  SQLite.

Переваги:

-         Файлова структура - вся база даних складається з одного файлу, тому її дуже легко переносити на різні машини.

-         Стандарти - хоча може здатися, що ця СУБД примітивна, але вона використовує SQL. Деякі особливості опущеними (RIGHT OUTER JOIN або FOR EACH STATEMENT), але основні все-таки підтримуються.

-         Відмінна при розробці та тестуванні - в процесі розробки додатків часто з'являється необхідність масштабування. SQLite пропонує все що необхідно для цих цілей, так як складається всього з одного файлу і бібліотеки написаної на мові C.

Недоліки:

-         Відсутність системи користувачів - більші СУБД включають в свій склад системи управління правами доступу користувачів. Зазвичай застосування цієї функції не так критично, так як ця СУБД використовується в невеликих додатках.

-         Відсутність можливості збільшення продуктивності - знову, виходячи з проектування, досить складно вичавити щось більш продуктивне з цієї СУБД.

2.     Оператори та  константи  SQLite.

Оператори:

-         У SQLite оператори порівняння двох значень можуть бути записані в 2 варіантах. Рівність може бути записано як = так і ==. А нерівність може бути записано! = І <>.

-         Оператор || - означає конкатенацію (об'єднання) рядків. При цьому, якщо одне із значень число, воно буде перетворено в текст і об'єднано з іншим значенням. Наприклад, select 1 || 1 поверне 11.

-         Оператор % - означає визначення залишку від ділення одного значення на інше. Наприклад, select 8% 3 & поверне 2.

-         Оператори IS і IS NOT - означають те ж саме, що і оператори = і! = За винятком випадків, коли один або обидва значення NULL. В цьому випадку якщо обидва значення NULL, тоді IS поверне 1 (істина), а IS NOT поверне 0 (брехня). Якщо одне із значень NULL а інше відмінне від NULL, тоді IS поверне 0 (брехня), а IS NOT поверне 1 (істина). Ці оператори використовуються для порівняння значень з NULL.

-         Оператор LIKE - здійснює порівняння за шаблоном. Значення праворуч від оператора LIKE повинно містити шаблон, а значення зліва - рядок, яка буде порівнюватися. Символ відсотка (%) в шаблоні означає збіг з будь-якою кількістю (в т.ч. і нуль) символів. Символ нижнє підкреслення (_) - означає один будь-який символ. Зауваження: оператор LIKE розрізняє верхній і нижній регістр тільки для ASCII символів. Це означає, що 'z' LIKE 'Z' - істина, а «я» LIKE 'Я' - брехня. Оператор LIKE можна зробити чутливим до регістру використавши PRAGMA case_sensitive_like

-         Оператор GLOB - аналогічний оператору LIKE, але використовує синтаксис як функція glob в Unix.

-         Оператор BETWEEN логічно еквівалентний двом парним порівнянь. Вираз "x BETWEEN y AND z" буде еквівалентно "x> = y AND x <= z" за винятком тільки того, що значення x буде обчислюватися тільки один раз.

Константи:

-         SQLITE3_ASSOC (integer)

Вказує на те, що Sqlite3Result :: fetchArray () метод повинен повернути масив, індексований по імені стовпця в відповідний набір результатів.

-         SQLITE3_NUM (integer)

Вказує на те, що Sqlite3Result :: fetchArray () метод повинен повернути масив, індексований за номером стовпця в відповідний набір результатів, починаючи з стовпця 0.

-         SQLITE3_BOTH (integer)

Вказує на те, що Sqlite3Result :: fetchArray () метод повинен повернути масив, індексований по імені стовпця і його номеру в відповідний набір результатів, починаючи з стовпця 0.

-         SQLITE3_INTEGER (integer)

Являє клас зберігання типу SQLite3 INTEGER.

-         SQLITE3_FLOAT (integer)

Являє клас зберігання типу SQLite3 REAL (FLOAT).

-         SQLITE3_TEXT (integer)

Являє клас зберігання типу SQLite3 TEXT.

-         SQLITE3_BLOB (integer)

Являє клас зберігання типу SQLite3 BLOB.

-         SQLITE3_NULL (integer)

Являє клас зберігання типу SQLite3 NULL.

-         SQLITE3_OPEN_READONLY (integer)

Вказує на те, що база даних SQLite3 доступна тільки для читання.

-         SQLITE3_OPEN_READWRITE (integer)

Вказує на те, що база даних SQLite3 доступна як для читання, так і для запису.

-         SQLITE3_OPEN_CREATE (integer)

Вказує на те, що файл бази даних SQLite3 буде створений в тому випадку, якщо він ще не існує.

-         SQLITE3_DETERMINISTIC (integer)

Вказує, що функція створена SQLite3 :: sqliteCreateFunction () детермінованість. Тобто вона завжди повертає однаковий результат для для однакових вхідних даних в одному вираженні SQL.

3.     Як отримати  значення ячейки з таблиці в SQLite.

-         За допомоги команди SELECT `назва_ячейки` FROM `назва_таблиці`;

4.     Як використовувати  SQLite.

-         Спосіб підключення до бази даних SQLite у всіх мовах практично однаковий. Всі вони вимагають для цього виконати два кроки: по-перше, включити в код бібліотеку SQLite (яка надає універсальні функції підключення), і по-друге, підключитися до бази даних і запам'ятати це підключення для подальшого використання. У PHP для цього служить бібліотека php-sqlite (php5-sqlite) (php5-sqlite).

 

5.     Як  встановлюються обмеження  під час  компіляції  бібліотеки SQLite.

-         Під час компіляції бібліотеки SQLite встановлюються наступні обмеження, які можна, при гострій необхідності, збільшувати:

6.     Угруповання  полів  SQLite.

-         Угруповання даних при вибірці даних реалізується за допомогою команди SELECT, що уточнює фрази GROUP BY і функцій агрегації.

7.     Особливості  SQLite.

-       SQLite не вимагає окремого процесу сервера або системи для роботи (без сервера).

-       SQLite поставляється з нульовою конфігурацією, що означає відсутність необхідності в налаштуванні або адмініструванні.

-       Повна база даних SQLite зберігається в одному крос-платформенном диску.

-       SQLite дуже маленький і легкий, менш 400KiB повністю сконфігурованих-менш 250KiB з додатковими функціями, опущеними.

-       SQLite є автономним, що означає відсутність зовнішніх залежностей.

-       SQLite-транзакції повністю сумісні з ACID, забезпечуючи безпечний доступ до декільком процесам або потокам.

-       SQLite підтримує більшість функцій мови запитів, знайдених в стандарті SQL92 (SQL2).

-       SQLite написаний на ANSI-C і забезпечує легкий і простий у використанні API.

-       SQLite доступний в UNIX (Linux, Mac OS-X, Android, iOS) і Windows (Win32, WinCE, WinRT).

8.     Виконання  основних операцій роботи з SQLite (створення бази і таблиць).

-         Бази даних створюються за допомоги команди CREATE DATABASE `назва_бази_даних`;

-         Таблиці створюються за допомоги команди CREATE TABLE `назва_таблиці`;

9.     Виконання  основних операцій роботи з  SQLite (вставка, оновлення, видалення).

-         Вставка виконується за допомоги команди INSERT INTO `назва_таблиці` (`колонка1`, `колонка2`,…`колонкаN`) VALUES (значення1, значення2,…значенняN);

-         Оновлення виконується за допомоги команди UPDATE `назва_таблиці` SET `колонка1`=значення1, `колонка2`=значення2,…`колонкаN`=значенняN;

-         Видалення виконується за допомоги команди DELETE FROM `назва_таблиці`;

10.  Виконання  основних операцій роботи з  SQLite (вибірка даних).

-         Вибірка виконується за допомоги команди SELECT `колонка1`, `колонка2`,…`колонкаN` FROM `назва_таблиці`;

    //Метод, отвечающий за регистрацию

    function registration() {

         if (isset($_POST['submit'])) {

                  if (isset($_POST['login']) && isset($_POST['email']) && isset($_POST['pass']) && isset($_POST['firstName']) && isset($_POST['secondName']) && isset($_POST['birthDay'])) {

                           if (strlen($_POST['login']) > 5 && strlen($_POST['email']) > 5 && strlen($_POST['pass']) > 5) {

 

                                   $stmt = $this->prepare('SELECT `login`, `email` FROM `User` WHERE `login`=:log OR `email`=:em');

 

                                   $stmt->bindValue(':log', $_POST['login']);

                                   $stmt->bindValue(':em', $_POST['email']);

                                  

                                   $result = $stmt->execute();

 

                                   $row = array();

 

                                   $i = 0;

 

                                   while($res = $result->fetchArray(SQLITE3_ASSOC)) {

                                            if (!isset($res['login'])) continue;

 

                                            $row[$i]['login'] = $res['login'];

                                            $row[$i]['email'] = $res['email'];

 

                                            $i++;

                                   }

 

                                   if (count($row) != 0) {

                                            $this->failMsg = "Пользователь с таким логином или електронной почтой уже существует!";

                                   } else {

 

                                            try {

                                                    if (strlen($_POST['birthDay']) != 12) {

                                                             throw new WrongDateException();

                                                    }

                                            } catch (WrongDateException $ex) {

                                                     $this->failMsg = "Неверный формат даты!";

                                                     return;

                                            }

                                            $stmtInsert = $this->prepare('INSERT INTO `User`(`login`,`email`,`password`,`firstName`,`secondName`,`regDate`,`birthDay`,`role`) VALUES (:log,:em,:pass,:fName,:sName,DATE("now"),:bday,0)');

 

                                            $stmtInsert->bindValue(':log', $_POST['login'], SQLITE3_TEXT);

                                            $stmtInsert->bindValue(':em', $_POST['email'], SQLITE3_TEXT);

                                            $stmtInsert->bindValue(':pass', $_POST['pass'], SQLITE3_TEXT);

                                            $stmtInsert->bindValue(':fName', $_POST['firstName'], SQLITE3_TEXT);

                                            $stmtInsert->bindValue(':sName', $_POST['secondName'], SQLITE3_TEXT);

                                            $stmtInsert->bindValue(':bday', $_POST['birthDay'], SQLITE3_TEXT);

 

                                            $stmtInsert->execute();

                                            $this->failMsg = true;

                                   }

                           }

                  }

             }

    }

 

    //Метод, отвечающий за логин

    function login() {

         if (isset($_POST['submit'])) {

                  $stmt = $this->prepare('SELECT `userID`,`login`, `password` FROM `User` WHERE `login`=:log AND `password`=:pass');

 

                  $stmt->bindValue(':log', $_POST['login']);

                  $stmt->bindValue(':pass', $_POST['pass']);

 

                  $result = $stmt->execute();

 

                  $row = array();

 

                  $i = 0;

 

                  while($res = $result->fetchArray(SQLITE3_ASSOC)) {

                          if (!isset($res['login']))

                                   continue;

 

                          $row[$i]['userID'] = $res['userID'];

                          $row[$i]['login'] = $res['login'];

                          $row[$i]['password'] = $res['password'];

 

                          $i++;

                  }

 

                  if (count($row) == 1) {

                          session_start();

                          $_SESSION['login_user'] = $row[0]['login'];

                          header("Location: index.php");

                  }

                  else {

                          $this->failMsg = "�'веден неверный логин или пароль!";

                  }

         }

    }

 

    //Метод, отвечающий за выход из аккаунта

    function logout() {

         unset($_SESSION['login_user']);

    }

 

    //Метод, получающий информацию о пользователе

    function getUserInfo() {

         if(isset($_SESSION['login_user']) && strlen($_SESSION['login_user']) >= 5) {

                  $stmt = $this->prepare('SELECT * FROM `User` WHERE `login`=:log');

 

                  $stmt->bindValue(':log', $_SESSION['login_user']);

 

                  $result = $stmt->execute();

 

                  $row = array();

 

                  $i = 0;

 

                  while($res = $result->fetchArray(SQLITE3_ASSOC)) {

                          if (!isset($res['login']))

                                   continue;

 

                          $row[$i]['userID'] = $res['userID'];

                          $row[$i]['email'] = $res['email'];

                          $row[$i]['birthDay'] = $res['birthDay'];

                          $row[$i]['firstName'] = $res['firstName'];

                          $row[$i]['secondName'] = $res['secondName'];

                          $row[$i]['regDate'] = $res['regDate'];

 

                          $i++;

                  }

 

                  if (count($row) == 1) {

                           $_SESSION['userID'] = $row[0]['userID'];

                           $_SESSION['email'] = $row[0]['email'];

                           $_SESSION['birthDay'] = $row[0]['birthDay'];

                           $_SESSION['firstName'] = $row[0]['firstName'];

                           $_SESSION['secondName'] = $row[0]['secondName'];

                           $_SESSION['regDate'] = $row[0]['regDate'];

                  } else {

                           $this->failMsg = "Произошла ошибка!";

                  }

         }

         else

                  header("Location: index.php");

    }

         $stmt = $this->prepare('SELECT `userID`,`firstName`,`secondName` FROM `User` WHERE `role`=1');

 

         $result = $stmt->execute();

    //Метод, создающий новую запись к врачу

    function newAppointment() {

         if(isset($_SESSION['userID']) && isset($_POST['submitApp'])) {

                  $stmt = $this->prepare('INSERT INTO `Appointment`(`doctorID`, `patientID`,`appDate`,`appMessage`,`status`) VALUES (:dID, :pID, :appDate, :appMsg, 0)');

 

                  $stmt->bindValue(':dID', $_POST['doctorID']);

                  $stmt->bindValue(':pID', $_SESSION['userID']);

                  $stmt->bindValue(':appDate', $_POST['date']);

                  $stmt->bindValue(':appMsg', $_POST['message']);

 

                  $result = $stmt->execute();

         }

    }

 

    //Метод, возвращающий историю посещений

    function getMyHistory() {

         if(isset($_SESSION['userID'])) {

                  $stmt = $this->prepare('SELECT `Appointment`.`appID`, `User`.`firstName`, `User`.`secondName`, `Appointment`.`appDate`, `Appointment`.`appMessage`, `Appointment`.`status` FROM `Appointment`, `User` WHERE `Appointment`.`doctorID` = `User`.`userID` AND `User`.`role` = 1 AND `Appointment`.`patientID` = :patient ORDER BY `Appointment`.`appDate`');

 

                  $stmt->bindValue(':patient', $_SESSION['userID']);

 

                  $result = $stmt->execute();

 

                  $row = array();

 

                  $i = 0;

 

                  while($res = $result->fetchArray(SQLITE3_ASSOC)) {

                          if(!isset($res['appID']))

                                   continue;

 

                          $row[$i]['appID'] = $res['appID'];

                          $row[$i]['firstName'] = $res['firstName'];

                          $row[$i]['secondName'] = $res['secondName'];

                          $row[$i]['appDate'] = $res['appDate'];

                          $row[$i]['appMessage'] = $res['appMessage'];

                          $row[$i]['status'] = $res['status'];

 

                          $i++;

                  }

 

                  return $row;

         }

    }
